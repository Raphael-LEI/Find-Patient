# 🎯 医溯 (Find Patient) 
### —— 一分钟找到你想研究的患者

**定位**：专注于非结构化临床文本挖掘的轻量级、自助式临床科研智能检索工具。它不替代医院现有的结构化查询系统，而是其**关键补充**。

---

## 🌟 核心价值：填补非结构化文本挖掘的空白

在临床科研领域，高达 **80%** 的关键科研信息蕴藏在自由文本中。然而，现有工具体系存在显著断层：
* **通用 BI 与 SQL**：无法理解语义，难以处理高度非结构化的医疗文本。
* **国际级系统 (如 SemEHR)**：虽然技术先进，但对中文医学叙事习惯和国内病历模板缺乏深度理解。
* **垂直领域工具 (如 PIRO)**：功能局限于单一体裁（如病理报告），无法覆盖更通用的病程记录。
* **重型基础设施 (如传统 CDR)**：部署周期以年计、成本达百万级，且操作复杂，无法满足研究者敏捷、自助的即时需求。

**医溯 (Find Patient)** 应运而生。它比 SemEHR 更懂中文病历、比 PIRO 应用范围更广、比传统 CDR 平台轻便易用数十倍。

## 🚀 核心逻辑与演进路径

### 1. 现状：极简搜索体验
提供类“百度搜索”的极简体验：研究者只需使用自然语言描述其想法（如“查找服用华法林后出现消化道出血的房颤老年患者”），系统即可在数分钟内返回精准结果。

### 2. 近期规划：验证与试点
* **独立版本**：以网页工具形式，通过手动上传脱敏文本文件快速验证核心价值。
* **Pilot 阶段**：与标杆医院合作，通过 CDR 安全接口实现半自动对接，采用按年订阅的科室级服务模式。

### 3. 长期愿景：智能科研入口
从单点工具演进为医院科研平台的智能模块。未来将采用 **Streamlit** 构建界面，基于 **Huatuo (华佗)** 等中文医学大模型与 **ChromaDB** 向量数据库实现语义检索，通过 **Docker** 容器化保障院内私有化部署。
